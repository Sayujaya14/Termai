# TOOLS.md - Local Notes

Skills define how tools work. This file is for your specifics — the stuff that's unique to this user's setup.

## What Goes Here

Things like:

- Preferred Python venv or conda env paths
- Dataset locations and naming conventions
- SSH hosts and aliases
- API endpoints or local service URLs
- Output format preferences (HTML vs PDF, chart styles)
- Anything environment-specific

## Examples

```markdown
### Datasets

- ~/data/ — default folder for CSV uploads
- iris.csv — sample dataset for EDA demos

### Conventions

- EDA reports → eda_report.html + PNG plots in task workspace
- Prefer seaborn over raw matplotlib when both work
```

## Why Separate?

Skills are shared across users. Your setup is yours. Keeping them apart means you can update skills without losing your notes.
